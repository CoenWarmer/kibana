`@kbn/eslint-plugin-eui-a11y` is an ESLint plugin providing custom rules for validating JSXCode in the Kibana repo for accessibility (a11y) (i.e. aria) tags on Eui components.

## `@kbn/eui-a11y/eui_elements_should_have_aria_label_or_aria_labelledby_props`

This rule warns engineers to add `aria-label` to certain EUI components. It attempts to infer a reasonable suggestion depending on the value and context of the element and then turns this into a i18n value.

Example:

```
/* file is in `x-pack/solutions/observability/plugins/apm/public/components/fleet_integration/apm_agents/runtime_attachment/discovery_rule.tsx` */

<EuiButtonIcon
    data-test-subj="apmDiscoveryRuleButton"
    iconType="pencil"
    color="primary"
    onClick={() => {
        onEdit(id);
    }}
/>
```

turns into:

```
<EuiButtonIcon
    aria-label={i18n.translate('xpack.apm.discoveryRule.button.ariaLabel', {
        defaultMessage: 'Button',
    })}
    data-test-subj="apmDiscoveryRuleButton"
    iconType="pencil"
    color="primary"
    onClick={() => {
        onEdit(id);
    }}
/>
```

It currently kicks in on some EUI components which reasonably need an `aria-label` (`EuiButtonIcon`, `EuiButtonEmpty`, `EuiSelect`, `EuiSuperSelect`, etc), but can be expanded with more.
